# HomeManager Inventory API Skeleton — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold the base structure of a .NET 10 REST API for HomeManager (Inventory module only) with Clean Architecture in 4 layers, EF Core + SQL Server, Swagger, and a tests project — with no business logic.

**Architecture:** Clean Architecture: Domain (entities/interfaces) ← Application (DTOs/services/validators) ← Infrastructure (EF + repositories) ← API (controllers/Swagger). Dependencies point inward. Repository + Unit of Work pattern with AutoMapper and FluentValidation.

**Tech Stack:** .NET 10, EF Core 10 + SQL Server, AutoMapper, FluentValidation, Swashbuckle (Swagger), xUnit + Moq + FluentAssertions.
